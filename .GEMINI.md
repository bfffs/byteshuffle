# GEMINI.md - byteshuffle

## Project Overview
`byteshuffle` is a Rust library providing SIMD-accelerated byte shuffle and unshuffle routines. Its primary goal is to improve the compressibility of data consisting of arrays of fixed-size objects by grouping bytes of the same significance together. This often creates long runs of identical bytes, which are highly efficient for general-purpose compression algorithms (like `gzip`, `lz4`, or `zstd`).

The project is a clean reimplementation of the shuffle filter from the [C-Blosc](https://www.blosc.org) library.

### Core Technologies
- **Rust:** Edition 2021, MSRV 1.89.0.
- **SIMD Intrinsics:** Specialized implementations for x86 architectures using **SSE2**, **AVX2**, and **AVX512F** (specifically AVX512F + AVX512BW).
- **Runtime Dispatch:** Uses the `ctor` crate to automatically execute the implementation selection logic before `main`. The actual CPU feature detection is performed by `std::is_x86_feature_detected!`.
- **Generic Fallback:** A portable, non-optimized implementation for non-x86 architectures or when SIMD is unavailable.

## Building and Running

### Key Commands
- **Build:** `cargo build`
- **Test:** `cargo test --all-targets` (This includes benchmarks in test mode to ensure they aren't broken).
- **Benchmark:** `cargo bench`
- **Documentation:** `cargo doc --no-deps`
- **Linting:** `cargo clippy --all-targets -- -D warnings`
- **Nightly Features:** Use `cargo build --features nightly` to enable experimental features like `BorrowedBuf` support.

### Environment Variables
- `RUSTFLAGS="-Dwarnings"`: Used in CI to treat all warnings as errors.

## Development Conventions

### Architecture
- **Runtime Selection:** Implementation selection logic resides in `src/lib.rs` under `select_implementation`.
- **Surgical Optimizations:** SIMD code is isolated in `src/sse2.rs`, `src/avx2.rs`, and `src/avx512f.rs`.
- **Testing Strategy:** Uses `rstest` for parameterized testing and `rand` for differential testing (comparing optimized implementations against the generic fallback).

### Coding Style
- **Formatting:** Strictly follows the configuration in `rustfmt.toml`.
- **Safety:** Extensive use of `unsafe` for SIMD intrinsics and raw pointer manipulation. Each `unsafe` block should ideally have a comment justifying its safety.
- **Linting:** Clippy is used for static analysis. Some specific lints (like `needless_range_loop` in SIMD routines) are occasionally allowed where they conflict with the author's preference for clarity in bit-manipulation logic.

### CI/CD
- **Cirrus CI:** Configured in `.cirrus.yml`, testing against MSRV, stable (x86_64 and aarch64), and nightly toolchains.
